# Пошаговая установка локальной ИИ — делаем вместе

## Часть 1: Ollama (чат — Llama, Mistral и др.)

### Шаг 1. Скачать Ollama
1. Открой в браузере: **https://ollama.com/download**
2. Найди блок **Windows**.
3. Нажми **Download** — скачается файл `OllamaSetup.exe` (примерно 50–100 МБ).

### Шаг 2. Установить
1. Запусти **OllamaSetup.exe** (двойной клик).
2. Если Windows спросит разрешение — нажми **Да**.
3. Дождись окончания установки (минута–две).
4. В конце может предложить открыть Ollama — можно нажать **Finish** или закрыть окно.

### Шаг 3. Проверить, что Ollama запущен
1. Открой **PowerShell** или **cmd** (Win+R → введи `powershell` → Enter).
2. Выполни команду:
   ```
   ollama --version
   ```
   Должна появиться строка с версией (например `ollama version is 0.x.x`).
3. Ещё раз выполни:
   ```
   ollama list
   ```
   Пока список пустой — это нормально. Модели скачаем в следующем шаге.

### Шаг 4. Скачать модель — какую выбрать

**Топ по «уму» (уровень рассуждений ближе к GPT-5 / Claude):**

| Модель | Команда | Нужно RAM | Что даёт |
|--------|--------|-----------|----------|
| **DeepSeek R1 70B** | `ollama pull deepseek-r1:70b` | **~48–64 GB** | Лучшая рассуждения, код, логика (как O3/Gemini Pro) |
| **Llama 3.3 70B** | `ollama pull llama3.3:70b-instruct` | **~48–64 GB** | Сильная общая модель, длинный контекст |
| **Qwen 2.5 72B** | `ollama pull qwen2.5:72b-instruct` | **~48–64 GB** | Очень умная, отличные рассуждения |

**Если у тебя 16–32 GB RAM** (обычный ПК) — бери одну из этих (быстрее и всё равно умные):

| Модель | Команда | RAM |
|--------|--------|-----|
| **Qwen 2.5 32B** | `ollama pull qwen2.5:32b-instruct` | ~20–24 GB |
| **Llama 3.2 11B** | `ollama pull llama3.2:11b-instruct` | ~8–12 GB |
| **Llama 3.2 3B** | `ollama pull llama3.2:3b-instruct` | ~4–6 GB |

**Что запустить сейчас (одну команду):**

- **Память 32 GB и больше** — ставим самую умную:
  ```
  ollama pull qwen2.5:72b-instruct
  ```
  или чуть легче:
  ```
  ollama pull deepseek-r1:70b
  ```

- **Память 16–24 GB** — ставим сильную, но реалистичную:
  ```
  ollama pull qwen2.5:32b-instruct
  ```
  Или из папки проекта: `.\ollama_pull_qwen32.ps1`

- **Память 8–16 GB** — ставим быструю и умную:
  ```
  ollama pull llama3.2:11b-instruct
  ```

Скопируй нужную команду в терминал и дождись окончания загрузки. Потом в JARVIS в списке моделей выбери соответствующую (Ollama — Qwen 2.5 72B / DeepSeek R1 / Llama 3.2 11B и т.д. — как добавим ниже).

**Ещё вариант — Huihui AI** (много открытых моделей, в т.ч. «abliterated» — облегчённые/оптимизированные версии):
- На Ollama они идут под префиксом `huihui_ai/`. Примеры:
  - `ollama pull huihui_ai/deepseek-r1-abliterated` — рассуждения, уровень o1
  - `ollama pull huihui_ai/qwen3-abliterated` — последние Qwen
  - `ollama pull huihui_ai/gemma3-abliterated` — хорошо тянется на одной видеокарте
- В JARVIS можно выбрать модель из блока «Ollama — Huihui AI» (если добавлен в список).

### Шаг 5. Проверить модель
Выполни:
```
ollama list
```
В списке должна быть строка с названием модели (например `qwen2.5:72b-instruct` или `llama3.2:11b-instruct`).

**Быстрая проверка из папки проекта:** запусти `.\check_ollama.ps1` — скрипт проверит, что Ollama отвечает на порту 11434 и есть ли модель для чата.

### Шаг 6. Подключить к JARVIS
1. Открой проект JARVIS, запусти приложение.
2. Перейди на вкладку **AI** (полный чат).
3. Нажми кнопку **Авто** (рядом с «∞ Агент»).
4. В списке найди блок **── Локально (Ollama) ──**.
5. Выбери свою модель (например **Ollama — Qwen 2.5 32B (баланс)** если ставил qwen2.5:32b-instruct, или **Ollama — Llama 3.2 11B** для 11B/3B).
6. Закрой попап. Убедись, что Ollama запущен (значок в трее или `ollama list` в терминале). Напиши в чате любое сообщение — ответ должен приходить от локальной модели.

**Если видишь ошибку 404 или 500:** Ollama не был запущен или выбрана модель, которой нет в `ollama list`. Запусти Ollama, в JARVIS выбери **Ollama — Qwen 2.5 32B** и попробуй снова.

**Если в ответе сервера: "unable to allocate CUDA_Host buffer"** — не хватает видеопамяти GPU. Переведи Ollama на CPU: из папки проекта выполни `.\ollama_use_cpu.ps1`, затем выйди из Ollama (трей → Quit Ollama) и запусти Ollama снова. Чат будет работать на процессоре (медленнее, но стабильно).

**Готово.** Чат теперь работает через Ollama без ключей и платежей.

## Часть 3: Локальная генерация изображений (Fooocus) — позже

Когда захочешь ставить генерацию картинок на своём ПК:

### Шаг 1. Скачать Fooocus
1. Открой: **https://github.com/lllyasviel/Fooocus/releases**
2. В списке **Assets** найди архив для Windows, например:
   - `Fooocus_win64_2-1-xxx.7z` (если есть 7-Zip)
   - или `.zip`, если без 7-Zip
3. Скачай архив (несколько ГБ).

### Шаг 2. Распаковать
- Распакуй в папку, например `C:\Fooocus` (без кириллицы в пути).

### Шаг 3. Запустить
- Если есть **NVIDIA**: запусти `run_nvidia_gpu.bat`.
- Если только **CPU**: запусти `run_cpu.bat`.
- При первом запуске скачаются модели — подожди.
- Откроется окно Fooocus: вводишь текст — генерируется картинка.

Пока что генерация из JARVIS идёт через Replicate (/image). Подключение JARVIS к локальному Fooocus по API можно добавить отдельно, когда будешь готов.

## Если что-то пошло не так

- **«ollama не является внутренней или внешней командой»** — перезапусти терминал или перезагрузи ПК после установки Ollama.
- **Ошибка при запросе в JARVIS** — проверь, что в терминале `ollama list` показывает хотя бы одну модель и что ты выбрал в JARVIS модель из блока «Локально (Ollama)».
- **Медленно отвечает** — для больших моделей нужен нормальный объём RAM (8–16 ГБ). Модели `phi3` и `llama3.1:8b` легче и быстрее.

Начинай с **Части 1** — как выполнишь шаги 1–6, напиши, что получилось, и перейдём к следующему (доп. модели или Fooocus).
